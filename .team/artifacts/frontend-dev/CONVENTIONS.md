# Frontend Developer Conventions

- **Component naming**: {e.g., PascalCase, one component per file}
- **Folder structure**: {e.g., feature-based -- /features/auth/, /features/dashboard/}
- **State management**: {e.g., local state by default, global store only for shared state}
- **Styling approach**: {e.g., Tailwind utility classes, no inline styles}
- **Import order**: {e.g., external libs -> internal modules -> components -> styles}
- **Props pattern**: {e.g., destructured in function signature, typed with TypeScript interfaces}
- **Event handling**: {e.g., handleXxx naming for handlers, onXxx for props}
- **Responsive**: {e.g., mobile-first, breakpoints: sm/md/lg/xl}
