# Screen 06 — Emergency Exit Sheet

**Flow:** Active session → early exit
**Purpose:** Deliberate friction. Never guilt.

---

## Content

### Bottom Sheet

| Element | Copy |
|---------|------|
| **Elapsed message** | You started some water a while ago. |
| **Instruction** | Type EXIT to end early. |
| **Input placeholder** | _(Empty field. No placeholder text.)_ |
| **Cancel button** | Back to water |
| **Confirm button (appears after typing EXIT)** | End |

---

## Microcopy

| Scenario | Copy |
|----------|------|
| **User types something other than EXIT** | _(No error message. The "End" button simply doesn't appear until the input matches.)_ |
| **User types "exit" (lowercase)** | _(Accept it. Case-insensitive.)_ |
| **After confirming exit** | _(No "Are you sure?" double-confirmation. One deliberate action is enough.)_ |
| **After exiting** | _(Return to home screen. No message like "You left early." No stats about how much time was remaining. Clean return.)_ |

---

## Notes
- "You started some water a while ago." — vague on purpose. No "You have 47 minutes left!" The app doesn't count at you.
- "Type EXIT" provides friction without guilt. It is a speedbump, not a wall.
- No guilt language. No "You'll lose your streak!" No "Are you sure you want to give up?"
- "Back to water" is the primary action (larger, more prominent). "End" is secondary.
- After exiting, the app says nothing. No judgment. No recap. Just the home screen.
- The exit is always available. The app is a companion, not a warden.
